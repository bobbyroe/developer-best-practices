Introduction

If you want to learn more about building fully accessible web experiences, you’ve come to the right place.

The contents here are based on the W3C’s Web Content Accessibility Guidelines (WCAG 2.1). Our goal is to provide the simplest and most effective best practices on how to design, implement and test web applications with accessibility firmly in mind.

Setup

For creating web apps with accessibility in mind, you’ll need some specific tools at hand. Let’s go over how to get set up.

Knowledge

To begin developing accessible web apps, you need to understand how our customers use them, and how to optimize for their specific needs.

First thing we need to talk about is plain, traditional HTML. You may feel that being a serious senior full stack developer, this topic will have nothing new to offer. Turns out ‘though that most accessibility problems are the result of just bad HTML.

You’re probably accustomed to reading and using a web application on a traditional screen and control it with a mouse and keyboard. You may not realize that a surprisingly large group of customers rely on alternate devices to access the same information. So, get familiar with the fundamentals of those: how they work, how they are commonly used and what to pay attention to when providing support for them.

Before you read on, be sure to keep in mind that common knowledge of accessibility is full of misunderstandings, half-truths and technical oddities. This guide attempts to be fool proof and give the best answers to the most relevant questions.

  • Colors and contrast
    • Visual elements – text, icons, images and inputs need to be easily perceivable
  • Semantics and their importance for a11y

  • ARIA – when HTML is not enough
    • ARIA builds on HTML for use in web applications
  • Intro to keyboard only usage
    • Many customers cannot use a pointing device (like a mouse), so they rely on a keyboard. Thus your software must be fully functional with a keyboard only.
  • Intro to screen reader usage
    • Basics of interacting and reading websites and applications with a screen reader.
